## Ch2: テスト関数

### venv（仮想環境）
- `python -m venv venv` で作成
- `source venv/bin/activate` で有効化
- `deactivate` で無効化
- anacondaと共存時は `python -m pytest` で実行すること（pytestコマンドだとanaconda側が使われる）

### cardsパッケージ
- `pip install ./cards_proj` でローカルインストール
- CLIコマンド: `cards add`, `cards start`, `cards finish`, `cards delete`

### データクラスのシリアライズ
- `to_dict()` — オブジェクト → 辞書（`asdict(self)`）
- `from_dict(d)` — 辞書 → オブジェクト（`cls(**d)`でアンパック）
- `@classmethod` — インスタンスなしでクラスから直接呼べるメソッド

### pytest.raises — 例外テスト
- `with pytest.raises(TypeError):` — 例外が出ることを検証
- `match=正規表現` — エラーメッセージも検証（簡潔）
- `as exc_info` — 例外オブジェクトを取り出して自由に検証（柔軟）

### Arrange-Act-Assert（AAA）パターン
テスト関数を3つのステージに分ける構造化パターン。

| ステージ | AAA | BDD (Given-When-Then) | やること |
|---------|-----|----------------------|---------|
| 1 | Arrange（準備） | Given（前提） | テスト対象のオブジェクトやデータを用意 |
| 2 | Act（実行） | When（もし） | テスト対象の操作を実行 |
| 3 | Assert（検証） | Then（ならば） | 結果が期待どおりか検証 |

- Bill Wake が2001年にAAAパターンを命名
- Kent Beck がTDDの一部として普及
- BDDではGiven-When-Then（Ivan Moore考案、Dan North普及）
- **アサーションはテスト関数の最後に配置する**

```python
# 例: AAAパターンの適用
def test_to_dict():
    # Arrange（準備）
    c1 = Card("something", "brian", "todo", 123)

    # Act（実行）
    result = c1.to_dict()

    # Assert（検証）
    expected = {
        "summary": "something",
        "owner": "brian",
        "state": "todo",
        "id": 123,
    }
    assert result == expected
```
